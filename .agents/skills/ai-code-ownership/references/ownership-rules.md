# Ownership Rules

- Coding assistants continue the context and style they are shown.
- Poor input code often produces more poor code.
- The assistant may not see paired components, existing repositories, policies, validators, or architectural boundaries.
- Never accept generated code that creates a duplicate source of truth.
- Require security, validation, error handling, and logging review.
- Rewrite generated code until it feels native to the project.
- Understand where the code came from and why it is shaped that way.
- Use tests to prove the generated change integrates with surrounding flows.
